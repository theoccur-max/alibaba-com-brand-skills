## Link handling rule

Default to the most specific confirmed browseable link available in this order:
1. folder link
2. category link
3. entry GGS library link
4. direct file link only when the user explicitly asks for a direct file, or when no browseable folder or category link is available

When a user asks for a specific asset, return:
- the exact known asset title, folder name, or tag clue
- the category path
- the preferred browseable Done link from this file
- a short note about whether the link opens a folder or category

Never invent a direct asset link that is not present in this file.
If the exact asset is not yet confirmed, return the closest category link and explain how to navigate inside the library.
